Testing Priorities

This page is designed to test the final priorities of images in a web page. There are 4 image requests to show how the browser heuristics affect their priorities in different parts of the page. If the heuristics change then this test may need to be revisted. This page uses a long amount of text to trigger a Low priority for the images below-the-fold. The third image has had it's priority boosted by using the fetchpriority attribute.

Expected results
URIPriorityReason
test-image.png?1 not Low Above the fold
test-image.png?2 Low Below the fold
test-image.png?3 not Low Boosted due to fetchpriority
test-image.png?4 Low Below the fold

Initial Priority

To render a webpage the browser has to request all the resources needed, but in general it can't do that all at once. It could take up all the bandwidth & CPU and still not be enough, especially now days of the multi-megabyte pages. Instead, only a few requests and responses are in transit at anytime. So, the browser needs to order those requests so it can start showing the page as soon as possible. The order the resource requests are in the HTML is also a factor.

Type Priority
Beacon Lowest
Font High
Image Low
JavaScript Normal
OCSP Request Highest
Stylesheet Normal
